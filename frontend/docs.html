Введение

Система верификации рукописного текста "Кто я?" позволяет идентифицировать автора рукописного текста на основе предоставленных образцов почерка.

Система использует современные алгоритмы машинного обучения и компьютерного зрения для анализа уникальных характеристик почерка, таких как:

  • Наклон букв
  • Размер и пропорции символов
  • Давление при письме
  • Уникальные особенности написания отдельных букв
  • Пространственное расположение текста

Для достижения наилучших результатов рекомендуется предоставлять несколько образцов почерка (3-5 страниц) для каждого пользователя.

Преимущества

  • Точность до 95% при достаточном количестве образцов
  • Поддержка различных языков и алфавитов
  • Простое API для интеграции

Ограничения

  • Требует качественных изображений (300+ DPI)
  • Может ошибаться при анализе очень коротких текстов
  • Не работает с печатным текстом

Быстрый старт

1. Добавление пользователей

Перед началом работы необходимо добавить пользователей в систему с образцами их почерка.

Шаг 1

Соберите образцы почерка

Попросите пользователя написать несколько страниц текста (рекомендуется 3-5 страниц). Текст должен быть написан в обычных условиях, без специального старания изменить почерк.

Шаг 2

Отсканируйте или сфотографируйте текст

Используйте сканер с разрешением не менее 300 DPI или камеру с хорошим качеством. Избегайте бликов, теней и искажений.

Шаг 3

Загрузите в систему

Используйте веб-интерфейс или API для загрузки изображений и создания профиля пользователя. 

curl -X POST \
  -H 'accept: application/json' \
  -H 'X-API-Key: aecbcab1-72b6-44d6-a50e-7bdc6e8d5829' \
  -H 'Content-Type: multipart/form-data' \
  -F 'full_name=Иван Петров' \
  -F 'user_id=user123' \
  -F 'samples=@T1.png;type=image/png' \
  -F 'samples=@T2.png;type=image/png' \
  -F 'samples=@T3.png;type=image/png' \
  -F 'samples=@T4.png;type=image/png' \
  -F 'samples=@T5.png;type=image/png' \
  -F 'notes=' \
  https://api.kto-ya.ru/v1/users

2. Верификация текста

После добавления пользователей вы можете проверять новые образцы текста на принадлежность конкретному автору.

Шаг 1

Подготовьте текст для проверки

Получите изображение рукописного текста, автора которого нужно определить. Чем больше текст, тем точнее будет результат.

Шаг 2

Отправьте на проверку

Используйте веб-интерфейс или API для отправки изображения. 

curl -X POST \
  -H 'accept: application/json' \
  -H 'X-API-Key: aecbcab1-72b6-44d6-a50e-7bdc6e8d5829' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@L3.png;type=image/png'
  https://api.kto-ya.ru/v1/verify \
Шаг 3

Получите результаты

Система вернет возможного автора с оценкой вероятности совпадения. 

{
    "success": true,
    "requestId": "req_123456789",
    "results": [
        {
            "userId": "user_123",
            "user_info": [
                {
                  "full_name": "Иван Петров",
                  "notes": "Сотрудник ФСБ"
                }
            ],
            "confidence": 0.92
        }
    ]

Генерирует API ключ с данными для вашего пользования.

Параметр Тип Обязательный Описание

Алгоритмы

Как это работает

Наша система использует комбинацию традиционных методов анализа почерка и современных алгоритмов машинного обучения для достижения высокой точности верификации.

Feature Extraction

Извлечение более 50 характеристик почерка, включая геометрические, статистические и топологические особенности.

Neural Networks

Глубокие сверточные сети (CNN) для анализа целостного образа почерка и выявления сложных паттернов.

Random Forest

Ансамблевый метод для классификации на основе извлеченных характеристик с оценкой важности каждой из них.

Similarity Scoring

Композитная оценка сходства с учетом весов различных характеристик для каждого конкретного пользователя.

Best Practices

Сбор образцов

  • Используйте разные ручки (шариковая, гелевая) для сбора образцов

  • Включайте образцы, написанные в разное время суток

  • Собирайте образцы с разным содержанием (текст, цифры, подписи)

Качество изображений

Рекомендуется

  • Разрешение 300+ DPI
  • Ровное освещение без теней
  • Фон контрастный, но не отвлекающий

Не рекомендуется

  • Размытые или нечеткие изображения
  • Сильные искажения перспективы
  • Водяные знаки или наложения

Интеграция с API

Совет 1: Кэширование результатов

Сохраняйте requestId для повторного запроса результатов, чтобы избежать повторной обработки.

Совет 2: Пакетная загрузка

При добавлении новых пользователей загружайте образцы пачками по 5-10 для ускорения обработки.

Совет 3: Держите API ключи в защищенном месте

API ключи очень важны, их утеря может дать доступ злоумышлинникам.

FAQ

Мы рекомендуем предоставлять не менее 3-4 страниц рукописного текста для каждого пользователя. Чем больше образцов, тем точнее будет результат.

Наша система способна обнаруживать попытки намеренного изменения почерка с точностью около 60%. Однако при профессиональной имитации (например, графологами) точность снижается. В таких случаях мы рекомендуем использовать дополнительные методы аутентификации.
